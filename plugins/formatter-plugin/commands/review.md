# Code Review Command

最近の変更やユーザーが指定したファイルに対してコードレビューを実行します。

## レビュー項目

1. **コード品質**
   - コードの可読性
   - 命名規則の遵守
   - 複雑度の評価

2. **ベストプラクティス**
   - 言語固有のベストプラクティス
   - セキュリティの考慮事項
   - パフォーマンスの懸念

3. **潜在的な問題**
   - バグの可能性
   - エッジケースの処理
   - エラーハンドリング

4. **改善提案**
   - リファクタリングの機会
   - より効率的な実装方法
   - テストカバレッジの向上

## 出力形式

レビュー結果を以下の形式で提供してください：

- ファイル名と行番号
- 問題の重要度（高・中・低）
- 具体的な問題点
- 改善案

日本語で丁寧に説明し、建設的なフィードバックを心がけてください。
